## 3. 段落スタイル定義

### Normal (基本スタイル)

- 両端揃え (`jc: both`)
- 禁則処理あり (`widowControl: off`)
- カーニング: 2
- フォントサイズ: **10.5pt** (sz=21)
- szCs: 12pt (24)

### 使用中のスタイル

| スタイルID | 名前 | 配置 | 特徴 |
|-----------|------|------|------|
| `a` | Normal | 両端揃え | 基本スタイル 10.5pt |
| `a3` | Salutation (挨拶) | Normal継承 | |
| `a4` | Closing (結語) | **右揃え** | 「敬具」に使用 |
| `a5` | Note Heading (記) | **中央揃え** | 「－記－」に使用 |
| `ac` | List Paragraph | 左インデント 840 (400字分) | リスト項目に使用 |

## 4. 文書構造（レイアウトパターン）

```
┌─────────────────────────────────────────┐
│                              ${日付}│  ← 右揃え, 10.5pt, MS P明朝
│                                            │
│ ${宛先部署名}                          │  ← 左揃え, 14pt (sz=28)
│ ${宛先名}                                    │  ← 左揃え, 12pt (sz=24)
│                   ${差出人所属部署名}│  ← 右揃え, 10.5pt
│              　${差出人名}            │  ← 右揃え, ind left=5880
│                                            │
│    ̲${文書タイトル}        │  ← 中央, HGPゴシックM, 14pt, 下線
│                                            │
│   ${本文:挨拶}                             │  ← 11pt, firstLine=330
│   ${本文:要件}    │  ← 11pt, ind left=214
│                                       ${結語}  │  ← Closing style, 右揃え, 11pt
│                                            │
│                  －記－                      │  ← Note Heading, 中央揃え
│                                            │
│ ${要件内容}│  ← 番号付きリスト, numId=5

│                                       以上  │  ← 右揃え
└─────────────────────────────────────────┘
```

### 4-1. プレースホルダー

#### 4-1-1. 日付

- 形式: 「YYYY年M月D日」
- 例: 2024年11月24日

#### 4-1-2. 宛先部署名

- 形式
  - 組織の場合： 「${組織名} ${部署名}」
    - 例: 〇〇株式会社 総務部 総務ユニット
    - 例: 〇〇中学校 教務課

- 宛先組織と差出人組織が同じ場合は、組織名を省略する.  
- 部署名は可能な限り正式名称を使用する.
- 組織名を含めたときに宛先部署名が長くなる場合は、組織名の後で改行してもよい.

- 該当者宛の宛先例: 保護者 各位

#### 4-1-3. 宛先名

- 例: 〇〇さん, 〇〇様, 〇〇御中, 〇〇各位 etc..
- 役職名を含める場合は、氏名の前に配置すること.
  - 例: 部長 田中太郎 様
- 宛先に合わせて敬称を選択すること.

#### 4-1-4. 差出人所属部署名

- 形式
  - 組織の場合： 「${組織名} ${部署名}」
    - 例: 〇〇株式会社 〇〇部 〇〇課
    - 例: 〇〇中学校 教務課
- 宛先組織と差出人組織が同じ場合は、組織名を省略する.  
- 組織名を含めたときに差出人部署名全体がが長くなる場合は、組織名のみとすることも可.

#### 4-1-5. 差出人名

- 例: 田中 太郎
- 役職名を含める場合は、氏名の前に配置すること.
  - 例: 部長 田中太郎 様

#### 4-1-6. 文書タイトル

- 例: 借用開発機材送付の件
- 例: 会議開催のお知らせ, 〇〇会議のｇ案内 etc..

#### 4-1-7. 本文

- 挨拶文: 定型的な挨拶文を使用することが望ましい.
  - 例: お疲れ様です。, いつもお世話になっております。, 平素より大変お世話になっております。 etc..
  - 組織外宛の場合は、時候の挨拶を含めて、より丁寧な挨拶文を使用することが望ましい.
    - 例: 〇〇の候、平素より格別のご高配を賜り、厚く御礼申し上げます。, いつも大変お世話になっております。 etc..
- 要件文: 用件を簡潔に記載すること.

#### 4-1-8. 結語

日本語文書における定型的な結語を使用すること.

- 例: 敬具

#### 4-1-9. 要件内容

要件に合わせて日本語文書として標準的な形式を選択すること.  
何が含まれるかによって、以下の形式が考えられる.

- 箇条書き
- 番号なしリスト
- 番号ありリスト
  - 1. 2. 3. 
    - (1) (2) (3) ...
      - (ⅰ) (ⅱ) (ⅲ) ...

- 項目建て

## 6. 番号付きリスト定義 (numId=5)

- 形式: `(%1)` → (1), (2), (3)...
- 開始番号: 1
- 左インデント: 360 twips
- ぶら下げ: 360 twips
- タブによるカラム整列（品名・管理番号・数量を揃え）

- ヘッダータブ位置: center=4252, right=8504
